# MFChanger - FC온라인 미니페이스 변경 프로그램 요구사항 문서

## 1. 프로젝트 개요

### 1.1 프로그램명
**MFChanger** (Mini Face Changer)

### 1.2 목적
FC온라인에서 선수 미니페이스(이하 '미페') 이미지를 사용자가 원하는 이미지로 쉽게 교체할 수 있도록 자동화하는 데스크톱 GUI 프로그램.

### 1.3 해결하는 문제
기존 미페 변경 절차는 다음과 같이 번거롭다:
1. FC온라인 홈페이지에서 원하는 선수의 미페 이미지를 찾기
2. 이미지 URL에서 `p`로 시작하는 미페 ID 확인
3. 교체할 이미지를 해당 ID로 파일명 변경
4. FC온라인 설치 폴더 내 특정 경로를 직접 탐색
5. 이미지 파일을 해당 경로에 붙여넣기
6. 파일 속성을 읽기전용으로 설정

**MFChanger는 이 과정을 3단계로 축소한다:**
> 미페 선택 → 교체할 이미지 선택 → 버튼 클릭 → 완료

## 3. 핵심 기능 명세

### 3.1 선수 검색 및 미페 ID 조회

#### 3.1.1 데이터 소스
- **FC온라인 공식 데이터센터 API** 활용 (Nexon Open API 또는 스크래핑)
  - API 키 필요 여부 확인 후 방식 결정
  - 대안: tots.gg, FC온라인 데이터센터(fconline.nexon.com) 직접 파싱
- 선수 정보에는 선수명, 시즌, 스탯, 그리고 **미페 이미지 URL(pXXXXXXX 형식의 ID 포함)** 이 포함됨

#### 3.1.2 검색 기능
- 선수 이름으로 텍스트 검색 (한국어/영어 모두 지원)
- 검색 결과 목록 표시: 선수명, 소속팀, 시즌, 미페 썸네일
- 동명이인 또는 동일 선수의 여러 시즌 카드 모두 표시
- 결과 클릭 시 해당 선수의 미페 ID(`pXXXXXXX`) 자동 추출

#### 3.1.3 미페 미리보기
- 선수 선택 시 현재 FC온라인 공식 미페 이미지를 화면에 표시

---

### 3.2 교체 이미지 선택

- 파일 탐색기 열기 버튼으로 이미지 파일 선택 (jpg, png, bmp, webp 지원)
- 드래그 앤 드롭으로도 이미지 파일 지정 가능
- 선택한 이미지 미리보기 표시
- 이미지 해상도 정보 표시 (권장: 128×128 / 256×256 / 512×512)
- 정방형이 아닌 이미지 선택 시 경고 표시 (강제 변환 옵션 제공)

---

### 3.3 미페 교체 실행

#### 3.3.1 처리 순서
1. FC온라인 설치 경로 자동 탐지 (레지스트리 또는 기본 설치 경로 확인)
2. 미페 파일이 저장되는 서브 경로 자동 결정
3. 교체할 이미지를 미페 ID 파일명(`pXXXXXXX.png` 등)으로 복사
4. 복사된 파일에 **읽기전용** 속성 설정 (게임 패치 시 덮어쓰기 방지)
5. 성공/실패 결과 알림

#### 3.3.2 FC온라인 설치 경로 처리
- 자동 탐지 실패 시 사용자가 수동으로 경로 지정 가능
- 지정된 경로는 설정 파일에 저장 (재시작 시 유지)

#### 3.3.3 기존 미페 백업
- 교체 전 원본 미페 파일 자동 백업 (선택적 설정)
- 백업 폴더 위치 설정 가능
- 백업 복원 기능 제공

---

### 3.4 미페 목록 관리

- 사용자가 변경한 미페 목록을 앱 내에서 관리
- 각 항목: 선수명, 미페 ID, 교체 이미지 경로, 변경 날짜
- 개별 항목 복원(원본으로 되돌리기) 기능
- 전체 복원 기능

---

### 3.5 설정(Settings) 화면

| 설정 항목 | 설명 |
|-----------|------|
| FC온라인 설치 경로 | 자동 탐지 또는 수동 지정 |
| 백업 기능 활성화 | 교체 전 원본 파일 백업 여부 |
| 백업 저장 위치 | 백업 파일 저장 폴더 |
| 시작 시 업데이트 확인 | 앱 실행 시 새 버전 자동 확인 |
| 업데이트 수동 확인 | 버튼 클릭으로 즉시 확인 및 업데이트 진행 |
| 현재 버전 정보 | 설치된 버전 표시 |

---

### 3.6 자동 업데이트 시스템

#### 3.6.1 업데이트 확인
- 앱 시작 시 GitHub Releases API를 통해 최신 버전 확인
- 현재 버전과 비교하여 새 버전 존재 시 알림 배너 표시
- 설정 화면에서 수동으로 업데이트 확인 버튼 제공

#### 3.6.2 업데이트 진행
- 알림 클릭 또는 업데이트 버튼 클릭 시 릴리즈 다운로드 페이지 또는 인스톨러 자동 실행
- 업데이트 후 앱 재시작 처리

#### 3.6.3 버전 정보
- 앱 내 버전은 `version.py` 또는 `__version__` 변수로 관리
- GitHub Releases의 태그(`v1.0.0` 형식)와 비교

│                  │                           │
│  검색 결과 목록  │   선수명 / 시즌           │
│  ┌────────────┐  │   미페 ID: p1234567       │
│  │ 손흥민 EPL │  │                           │
│  │ 손흥민 UCL │  ├───────────────────────────┤
│  │ ...        │  │   [교체할 이미지 미리보기]│
│  └────────────┘  │                           │
│                  │   [이미지 파일 선택] 또는  │

│                  │                           │

│                  │   [    미페 교체 실행    ] │

### 4.2 UX 원칙
- **단순함 우선**: 핵심 기능(검색 → 선택 → 교체)이 메인 화면에서 모두 완결
- **즉각적인 피드백**: 검색, 파일 선택, 교체 완료 시 명확한 상태 표시
- **오류 방지**: 잘못된 이미지 비율, 경로 오류 등 사전 검증 및 안내
- **되돌리기 지원**: 교체 전 백업으로 실수 복구 가능

## 6. 미페 파일 관련 스펙

| 항목 | 내용 |
|------|------|
| 파일명 형식 | `p{7자리 숫자}.png` (예: `p1234567.png`) |
| 권장 해상도 | 128×128, 256×256, 512×512 (정방형) |
| 지원 포맷 | PNG (게임 내 사용), 입력은 JPG/PNG/BMP/WEBP 허용 후 변환 |
| 파일 속성 | 읽기전용 설정 필수 (게임 패치 덮어쓰기 방지) |
| 저장 경로 | FC온라인 설치 폴더 내 특정 서브 디렉토리 |

## 7. 배포 및 버전 관리

- **패키징**: PyInstaller로 단일 실행파일(`.exe`) 생성
- **배포**: GitHub Releases에 버전별 실행파일 업로드
- **버전 형식**: Semantic Versioning (`v주.부.수`, 예: `v1.0.0`)
- **업데이트 채널**: GitHub Releases API (`/repos/{owner}/{repo}/releases/latest`)

## 9. 미결정 사항 (추가 조사 필요)

1. **FC온라인 API 방식**: Nexon Open API 공개 키 발급 가능 여부, 미페 이미지 URL 패턴 확인
2. **미페 파일 저장 경로**: FC온라인 설치 폴더 내 정확한 서브 경로 확인 필요
3. **이미지 포맷**: 게임이 PNG 외 DDS 등 다른 포맷을 사용하는지 확인
4. **읽기전용 설정 방식**: `os.chmod` vs `ctypes`(Windows 전용) 방식 검토
